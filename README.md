# مدى (Mada) — واجهة المتجر الإلكتروني

واجهة أمامية كاملة لمتجر "مدى" مبنية بـ Next.js 16 (App Router, Turbopack) و React 19 و TypeScript و Tailwind CSS، بدعم كامل للعربية (RTL)، باستخدام `react-icons` للأيقونات و `zod` للتحقق من صحة النماذج.

> محتاجة Node.js 20 أو أحدث (متطلب Next.js 16).

## تشغيل المشروع

```bash
npm install
npm run dev
```

افتحي `http://localhost:3000`. (الـ dev server بيشتغل بـ Turbopack تلقائيًا، أسرع بكتير من قبل.)

## نظام التصميم

الألوان مُستخرجة فعليًا من اللوجو المرفق:

- **Sage (أخضر السيج)** `sage-500 #71896A` — اللون الأساسي للأزرار والتفاعل.
- **Clay (تان/طيني)** `clay-400 #C9925E` — لون التمييز (الأسعار، الشارات، الخصومات).
- **Cream (كريمي)** `#F7F4EC` — خلفية الموقع، نفس خلفية اللوجو.
- **Ink** `#2A2E26` — لون النص الأساسي (أخضر غامق بدل الأسود الفاحم).

الخطوط: **El Messiri** للعناوين (تحمل طابع خط اللوجو)، و **Tajawal** للنصوص (وضوح عالي للعربية). يتم تحميلهما عبر Google Fonts في `src/app/layout.tsx`.

العنصر المميز (Signature): خط متعرّج بنقطتين (`SectionDivider`) مستوحى من خط الكاريكلتشر في اللوجو وعجلات عربة التسوق المدمجة فيه، يُستخدم كفاصل بين أقسام الصفحة الرئيسية بدل الخط العادي.

## البنية

```
src/
  app/            صفحات Next.js (App Router)
  components/
    layout/       Header, Footer
    ui/           عناصر تصميم أساسية (Button, Input, ProductCard...)
    sections/     أقسام الصفحة الرئيسية
  context/        CartContext (سلة تسوق تعمل بـ localStorage)
  lib/
    types.ts      أنواع TypeScript (Product, Category...)
    data.ts       بيانات تجريبية (Mock) — استبدليها لاحقًا باستدعاءات حقيقية
    validations.ts  مخططات zod لكل النماذج
    utils.ts      أدوات مساعدة (تنسيق السعر...)
```

## ملاحظات مهمة قبل الربط بالباك إند

- البيانات في `src/lib/data.ts` بيانات تجريبية فقط. الدوال (`getProducts`, `getProductBySlug`...) مكتوبة بحيث يسهل استبدال محتواها لاحقًا باستدعاء قاعدة البيانات عبر Server Actions أو API Routes في Next.js، من غير ما تغيّري في باقي الواجهة.
- نماذج تسجيل الدخول والتسجيل والتواصل وإتمام الطلب بها تحقق كامل بـ `zod` على المتصفح، لكنها لسه مش متصلة بسيرفر حقيقي (فيه تعليق `// TODO` في كل نموذج في مكان الاتصال بالباك إند).
- سلة التسوق (`CartContext`) شغالة بالكامل على المتصفح (state + localStorage)، جاهزة إنها تتحول لاحقًا لتستخدم قاعدة بيانات بدل التخزين المحلي.

## الخطوة الجاية

بعد ما نخلص صفحات المتجر، الخطوة الجاية (حسب الاتفاق) هي عمل **Dashboard لوحة تحكم الـ admin** لإدارة المنتجات والطلبات والمستخدمين، وربط الـ backend بالكامل عبر Server بتاع Next.js.
